    run module¶

    Run clang-tidy and clang-format on a list of changed files provided by GitHub’s REST API. If executed from command-line, then main() is the entrypoint.

    Seealso

    • github rest API reference for pulls

    • github rest API reference for commits

    • github rest API reference for repos

    • github rest API reference for issues

    cpp_linter.run.run_clang_tidy(filename: str, file_obj: dict[str, Any], version: str, checks: str, lines_changed_only: int, database: str, repo_root: str, extra_args: list[str]) → None[source]¶

    Run clang-tidy on a certain file.

    Parameters:¶
    filename: str¶

    The name of the local file to run clang-tidy on.

    file_obj: dict[str, Any]¶

    JSON info about the file.

    version: str¶

    The version of clang-tidy to run.

    checks: str¶

    The str of comma-separated regulate expressions that describe the desired clang-tidy checks to be enabled/configured.

    lines_changed_only: int¶

    A flag that forces focus on only changes in the event’s diff info.

    database: str¶

    The path to the compilation database.

    repo_root: str¶

    The path to the repository root folder.

    extra_args: list[str]¶

    A list of extra arguments used by clang-tidy as compiler arguments.

    Note

    If the list is only 1 item long and there is a space in the first item, then the list is reformed from splitting the first item by whitespace characters.

    cpp-linter --extra-arg="-std=c++14 -Wall"

    is equivalent to

    cpp-linter --extra-arg=-std=c++14 --extra-arg=-Wall

    cpp_linter.run.make_annotations(style: str, file_annotations: bool, lines_changed_only: int) → int[source]¶

    Use github log commands to make annotations from clang-format and clang-tidy output.

    Parameters:¶
    style: str¶

    The chosen code style guidelines. The value ‘file’ is replaced with ‘custom style’.

    file_annotations: bool¶

    A flag that corresponds to the --file-annotations CLI option.

    lines_changed_only: int¶

    Corresponds to the --lines-changed-only CLI option.

    • 0 means all lines.

    • 1 means only lines in the diff chunks.

    • 2 means only lines in the diff with additions.

    Returns:¶

    A boolean describing if any annotations were made.
